## 📋 Deployment Steps

### Step 1: Upgrade Railway Plan (if needed)
- Free plan has reached resource limits
- **Options:**
  - Upgrade to **Hobby Plan** ($5/month) for unlimited projects
  - Or use **Render.com** or **Fly.io** (free tiers available)

### Step 2: Create Railway Project

1. Go to https://railway.app/dashboard
2. Click "+ New"
3. Select "Deploy from GitHub repo"
4. Authorize Railway with GitHub
5. Choose **hdr122/joniah-pharmacy**
6. Railway will auto-detect and deploy

### Step 3: Add PostgreSQL Database

1. In Railway project, click "+ Add Service"
2. Select **PostgreSQL**
3. Railway auto-generates `DATABASE_URL` environment variable
4. Database will be ready in ~2 minutes

### Step 4: Configure Environment Variables

In Railway dashboard, go to your project's "Variables" tab and add:

```bash
# Auto-generated by PostgreSQL service (do not manually set)
DATABASE_URL=postgresql://[auto-set by Railway]

# Security
JWT_SECRET=generate-a-random-32-char-string-here
OAUTH_SERVER_URL=https://oauth.manus.im
VITE_APP_ID=test-app-id
OWNER_OPEN_ID=test-owner-id
VITE_GOOGLE_MAPS_API_KEY=AIzaSyD_your_actual_maps_key_here

# Firebase Service Account (full JSON string)
FIREBASE_SERVICE_ACCOUNT='{"type":"service_account","project_id":"joniah-pharmacy",...}'

# Server
PORT=3000
NODE_ENV=production
```

**⚠️ IMPORTANT: Firebase credentials should be rotated from your `.env` file**
- Go to Firebase Console > Project Settings > Service Accounts
- Generate a NEW service account key
- Copy the full JSON and paste as the `FIREBASE_SERVICE_ACCOUNT` variable

### Step 5: Automatic Deployment

1. Railway **automatically links** PostgreSQL service to `DATABASE_URL`
2. Build command detected: `pnpm install && pnpm run build`
3. Start command detected: `node server/index.js`
4. **Auto-deploys** on every GitHub push to `main` branch

### Step 6: Monitor Deployment

1. Go to Railway project dashboard
2. Click "Deployments" to watch build progress
3. View real-time logs in the "Logs" tab
4. Once deployed, you'll get a public Railway app URL

## Common Issues and Solutions

### Build Failures
- Check the "Build Logs" tab in Railway
- Ensure all environment variables are set
- Verify the database connection string is correct

### Database Migration Errors
- SSH into the container via Railway CLI
- Run `pnpm run db:push` manually
- Check Drizzle ORM configuration in `server/_core/db.ts`

### Port Issues
- Railway automatically assigns a port via `PORT` environment variable
- The application listens on this port
- No need to manually configure port binding

## Database Backup

To backup the MySQL database on Railway:

```bash
# Using Railway CLI
railway run mysqldump -h $DATABASE_HOST -u $DATABASE_USER -p$DATABASE_PASSWORD $DATABASE_NAME > backup.sql
```

Or use Railway's built-in backup features in the dashboard.

## Environment Variables Reference

| Variable | Description | Value |
|----------|-------------|-------|
| `DATABASE_URL` | PostgreSQL connection (auto-set) | `postgresql://user:pass@host:5432/db` |
| `JWT_SECRET` | JWT signing secret | 32+ random characters |
| `OAUTH_SERVER_URL` | OAuth server URL | `https://oauth.manus.im` |
| `VITE_APP_ID` | Application ID | `test-app-id` |
| `OWNER_OPEN_ID` | Super admin OpenID | `test-owner-id` |
| `VITE_GOOGLE_MAPS_API_KEY` | Google Maps API key | From Google Cloud Console |
| `FIREBASE_SERVICE_ACCOUNT` | Firebase credentials (JSON) | Full service account JSON |
| `PORT` | Server port | `3000` |
| `NODE_ENV` | Environment | `production` |
