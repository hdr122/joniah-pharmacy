# 📦 دليل النسخ الاحتياطي والمزامنة مع GitHub

تم إعداد نظام نسخ احتياطي كامل ومزامنة ثنائية مع GitHub لحماية بيانات مشروع صيدلية جونيا.

---

## 🔗 معلومات المستودع

- **رابط GitHub:** https://github.com/harthhdr/joniah-pharmacy-backup
- **نوع المستودع:** خاص (Private)
- **الوصف:** نسخة احتياطية كاملة لمشروع صيدلية جونيا مع قاعدة البيانات

---

## 🚀 الأوامر المتاحة

### 1. المزامنة الثنائية مع GitHub
```bash
pnpm sync
```
**الوظيفة:**
- يفحص التغييرات المحلية ويحفظها تلقائياً
- يسحب أحدث التغييرات من GitHub
- يدفع جميع التغييرات إلى GitHub و Manus

**متى تستخدمه:**
- بعد إجراء تعديلات على الكود
- قبل البدء في العمل للحصول على آخر التحديثات
- بشكل دوري للحفاظ على التزامن

---

### 2. النسخ الاحتياطي لقاعدة البيانات
```bash
pnpm backup
```
**الوظيفة:**
- ينشئ نسخة احتياطية من schema قاعدة البيانات
- يحفظ معلومات النسخة الاحتياطية في مجلد `backups/`
- يسجل التاريخ والوقت لكل نسخة احتياطية

**متى تستخدمه:**
- قبل إجراء تغييرات كبيرة على قاعدة البيانات
- بشكل دوري (يومي/أسبوعي) حسب أهمية البيانات
- قبل ترقية النظام أو تحديث المكتبات

---

### 3. النسخ الاحتياطي التلقائي مع المزامنة
```bash
pnpm backup:auto
```
**الوظيفة:**
- ينفذ النسخ الاحتياطي لقاعدة البيانات
- يزامن جميع التغييرات مع GitHub تلقائياً
- يوفر حماية شاملة في خطوة واحدة

**متى تستخدمه:**
- في نهاية يوم العمل
- بعد إضافة ميزات جديدة
- كجزء من روتين الصيانة الدورية

## 🔐 الأمان

- **المستودع خاص:** جميع البيانات محمية ولا يمكن الوصول إليها إلا من قبل المالك
- **المصادقة:** تستخدم GitHub CLI المصادقة الآمنة عبر tokens
- **التشفير:** جميع الاتصالات مع GitHub مشفرة عبر HTTPS

## 🔄 المزامنة التلقائية

### الإعداد اليدوي (اختياري)
يمكنك إعداد مزامنة تلقائية باستخدام cron jobs:

```bash
# تحرير crontab
crontab -e

# إضافة مهمة للمزامنة كل ساعة
0 * * * * cd /home/ubuntu/joniah_pharmacy && pnpm sync

# إضافة مهمة للنسخ الاحتياطي الكامل يومياً في الساعة 2 صباحاً
0 2 * * * cd /home/ubuntu/joniah_pharmacy && pnpm backup:auto
```

## 🆘 استعادة البيانات

### استعادة من GitHub
```bash
# الانتقال إلى مجلد المشروع
cd /home/ubuntu/joniah_pharmacy

# سحب آخر نسخة من GitHub
git pull github main

# إذا كانت هناك تعارضات، يمكنك إعادة تعيين المشروع بالكامل
git fetch github
git reset --hard github/main
```

### استعادة نسخة احتياطية محددة
```bash
# عرض قائمة النسخ الاحتياطية المتاحة
ls -lh backups/

# استعراض محتوى نسخة احتياطية محددة
cat backups/backup-YYYY-MM-DD.json
```

## ⚠️ ملاحظات هامة

1. **لا تحذف مجلد `.git`** - يحتوي على سجل كامل للتغييرات
2. **احتفظ بنسخة من tokens** - في حالة فقدان الوصول إلى GitHub CLI
3. **راجع النسخ الاحتياطية بانتظام** - للتأكد من صحتها
